### 1. Visual Hierarchy

### 3. Actionability
- Always include "TEACH IT THIS WAY" section
- Provide specific alternative phrasing
- Include memory aids

### 4. Completeness (1-page limit)
- Error + refutation

Pastor planning sermon on Romans 1:16 ("power of God"):
1. Checks G1411 quick reference card

Small group leader preparing study on church (ekklesia):
1. Reviews G1577 quick reference card

3. Teaches "assembly/congregation" instead

Student writing exegesis paper:
1. Consults card before making word study claims
2. Verifies etymology vs. usage meaning
3. Cites proper authorities (BDAG not blog)
4. Avoids common pitfalls

### Scenario 4: Translation Consultant
Consultant reviewing translation notes:
1. Checks cards for target language words
2. Ensures translation notes don't perpetuate errors
3. Provides accurate guidance to translators
4. Prevents error propagation to new languages

## Production Workflow

### Per Word (20 minutes)
1. **Pull data from YAML** (2 min)
   - Error claim
   - Refutation
   - Correct teaching
   - Sources

2. **Write card content** (10 min)
   - Simplify for 1-page format
   - Create "TEACH IT THIS WAY" section
   - Write pastoral notes
   - Add memory aid

3. **Format card** (5 min)
   - Apply template
   - Check 1-page constraint
   - Add visual hierarchy
   - Generate QR code

4. **Review for clarity** (3 min)
   - Scannable?
   - Actionable?
   - Gracious tone?
   - Complete?

### Batch Production
- Generate 50 cards = 50 Ã— 20 min = 1,000 min = 17 hours
- Can be done by assistant/student after YAML complete
- Quality check by researcher: 2 min/card = 100 min
