# Numeral Classifiers

**Status:** Documentation complete
**TBTA Relevance:** High - affects Mayan (22 langs universally), widespread in Asian languages
**Greek/Hebrew:** Not present - numerals directly modify nouns

## Overview

Numeral classifier systems require a classifier morpheme to appear between a numeral and the noun being counted. Classifiers categorize objects by shape, function, animacy, and other semantic properties. This construction is absent from Greek and Hebrew, which use numerals directly with nouns.

## Basic Pattern

**Greek/Hebrew (no classifiers):**
- "three fish" = [NUM three] [NOUN fish]

**Classifier language:**
- "three fish" = [NUM three] [CLASSIFIER animal/fish] [NOUN fish]
- Classifier is obligatory
- Choice of classifier depends on semantic category

## Classifier Categories

### Shape-Based
- **Long objects:** sticks, ropes, snakes, roads
- **Flat objects:** paper, cloth, leaves, mats
- **Round objects:** fruits, balls, round stones
- **Cylindrical objects:** logs, pipes, cigarettes

### Function-Based
- **Handled objects:** tools, weapons, utensils
- **Containers:** vessels, boxes, bags
- **Vehicles:** cars, boats, carts
- **Buildings:** houses, temples, structures

### Animacy-Based
- **People/humans:** person classifier
- **Animals:** animal classifier (may subdivide)
- **Plants:** plant/tree classifier
- **Inanimate:** general object classifier

### Abstract
- **Events:** occasions, times, instances
- **Abstract concepts:** faith, hope (requires classifier!)
- **Temporal units:** days, months, years

## Mayan Numeral Classifiers

### K'iche' Examples (20+ classifiers)

**Shape classifiers:**
- `junpe q'a` (one + round classifier) - fruits, round objects
- `junp' tzutz` (one + completion classifier) - completed objects
- `junpik` (one + handle classifier) - handled objects, tools
- `oxche` (three + long classifier) - sticks, rope, long objects
- `oxp'ak` (three + flat classifier) - paper, cloth, flat objects

**Temporal:**
- Days, months, years require temporal unit classifiers

**Abstract:**
- Even abstract nouns require classifier choice

### Ch'ol Pattern
- Uses `-p'ej` as generic classifier with numerals 1-19
- More streamlined than K'iche' system
- Still obligatory for counting

### Universality in Mayan
**All 22 Mayan languages:**
- Have numeral classifier systems
- 250-500 distinct forms per language (when including positionals)
- Obligatory for quantification
- Deep integration with noun class systems

## Cultural Variations

### Metaphorical Weight

**"Body of Christ" (corporate unity):**
- If numbered ("one body"), requires classifier
- **Person-collective classifier?** Emphasizes individuals
- **Unity/completeness classifier?** Emphasizes oneness
- Choice affects how corporate unity is understood

**"Two or three gathered" (Matthew 18:20):**
- Small number of people
- Classifier choice:
  - **Person classifier** (individuals)
  - **Small-group classifier** (if available)
  - Affects sense of minimum community

### Sacred Object Categorization

**Temple items:**
- Classifiers assign items to categories
- Shape-based? Function-based? Sacred-status-based?
- Cultural perception of sacred objects embedded in classifier choice

**Resurrection bodies:**

- Affects materiality perception

## Recommendations

### For TBTA Annotation

1. **Mark all quantified nouns**
   - Even "one X" requires annotation
   - Include semantic category

2. **Create semantic taxonomy**
   - Shape categories (long, flat, round, etc.)
   - Function categories (tool, container, vehicle, etc.)
   - Animacy categories (human, animal, plant, inanimate)
   - Abstract categories (quality, event, state, etc.)

3. **Flag theologically significant quantifications**
   - Trinity, apostles, tribes, spiritual entities
   - Note ambiguity and implications

### For Tool Development

1. **Automatic semantic categorization**
   - Based on noun semantics
   - Suggest classifier type

2. **Theological notes for significant terms**
   - Pre-annotated classifier suggestions
   - Community discussion points

3. **Cross-reference actual translations**
   - What classifiers have Mayan translations used?
   - Document community decisions

### For Translation Practice

1. **Learn target language classifier system thoroughly**
   - Semantic categories may differ from expectations
   - Cultural categories embedded in classifiers

2. **Consult community on ambiguous cases**
   - Especially theologically significant terms
   - Classifier choice can affect doctrine

3. **Consistency across Scripture**
   - Use same classifier for same entities
   - Unless context requires different emphasis

4. **Don't create artificial categories**
   - Use existing classifier system
   - Don't import European categories
